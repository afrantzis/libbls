The libbless library uses a posix 1.c influenced style for its API. This means:

\startitemize

\item Every function must return an 'int' data object representing the function
	error code. The special error value 0 means that no error occured.

\item All values that are returned by a function are returned using output
	parameters (aka call by reference). These parameters preceed any other 
	parameters in the parameter list. The only exception is when the function
	is part of an ADT. In that case the ADT object is always the first
	parameter:
	
	\starttyping
	int stack_pop(stack_t *stack, void *val);
	\stoptyping

\stopitemize

\stoptext
